## 🚀 How It Works

### Sprint Cycle (Automated)
```
Every 10 minutes:
1. Scrum Master creates sprint issue
2. Dev Agent implements assigned stories
3. QA Agent verifies
4. PM Agent accepts
5. CI builds APK automatically
6. Retrospective captured
7. Next sprint begins
```

### Manual Triggers
```bash
# Start specific sprint
gh workflow run sprint-timer.yml -f sprint_number=3 -f sprint_goal="Environment Config"

# Check build status
gh run list

# View sprint issues
gh issue list --label "rapid-scrum"
```
